# CASA × AINET — TDA и поток динамики: operational playbook v1

**Purpose:** переводить [`kb-tda-flow-machines-ca-ainet-fundamentals-v1.md`](kb-tda-flow-machines-ca-ainet-fundamentals-v1.md) в **действия, вопросы к спецификации и проверки** при проектировании CA/NCA и (гипотетического) обмена между узлами.  
**Не заменяет:** курс по алгебраической топологии, музыкальный стек Sony CSL, черновики протокола ИИнет.

---

## 1. Когда открывать этот playbook

- Нужно **сравнивать паттерны** CA на разных сетках, после шумов или усечения поля.
- Спор: передаём ли мы между агентами **сырое поле** или **сжатое инвариантное** описание.
- Проектирование **продолжения симуляции** другим агентом («подхватил поток») без bitwise копирования.
- Связка с обучением NCA: дополнительные **метрики** к пиксельному loss — см. fundamentals и [`kb-distill-growing-neural-ca-fundamentals-v1.md`](kb-distill-growing-neural-ca-fundamentals-v1.md).

## 2. Fundamentals → вопросы к эксперименту

| Идея (fundamentals) | Вопрос к спецификации |
|----------------------|----------------------|
| **Фильтрация / высота** в TDA | Что задаёт комплекс точек или ячеек: порог по α, по норме скрытых каналов, время жизни компонента? |
| **Устойчивость инвариантов** | Какая дистанция между диаграммами считается «тот же класс паттерна» (bottleneck, Wasserstein, эвристика)? |
| **Динамика** | Признак по окну из \(k\) шагов: суммарный баркод, дельты, или только конец окна? |
| **Flow / траектория** | Пространство состояний для марковской/латентной модели — сырые клетки, PCA/UMAP подполе, топопризнаки, смесь? |
| **Стыковка узлов AINET** | Два узла могут договориться о **общем ключе версии** (схема фильтрации + шаг решётки) — зафиксировано ли явно при обмене? |

---

## 3. Мини-чеклист перед первым прототипом TDA на поле

1. Зафиксирован **размер/подвыборка** (иначе профилирование бессмысленно).
2. Один канал или смесь каналов для построения комплекса — **документировано**.
3. Есть **базовый sanity check**: два явно разных паттерна дают различимые дескрипторы; два близких после мелкого шума — близко по выбранной метрике.
4. Явно отмечено, что топопризнак **не** присвоен человекочитаемой семантике («это сортировка») без отдельной валидации на задаче.

---

## 4. Мини-чеклист для «продолжения потока» (мотив FM)

1. Определён **интервал дискретизации времени**: один шаг CA = один тик модели потока или пакет шагов.
2. Способ **стыковки начального состояния** второго агента задан (копия латента, затухание расхождения, проекция на аттрактор).
3. Критерий успеха количественный (напр. сохранение класса топопризнака на горизонте \(T\) шагов), а не только «визуально похоже».
