# Миграция: карточки проектов в scope (v1)

**Цель:** структурировать `scope-door-to-singularity` и `scope-portal` без смены `section_id`, чтобы **hot-context**, манифест `memory-architecture-v1.json` и потребители команд не требовали доработок движка.

## Правило

- Секции **`scope-door-to-singularity`** и **`scope-portal`** остаются с теми же id.
- Детализация группируется заголовками **`### Карточка: …`** внутри секции.
- Все прежние буллеты переносятся без удаления смысла (допускается группировка под новый `###`).

## Проверка «ничего не потеряли»

1. На ветке канона: `git diff main...HEAD -- agent-notes.md`
2. Для scope-блоков: сравнить **конкатенацию строк-буллетов** до/после (игнорируя заголовки `##` / `###` и строки «Формат:»).
3. Опционально: `grep -n "scope-door-to-singularity\|scope-portal" agent-notes.md` — границы секций те же.
4. После merge: `memory_health` и `read_hot_context` с `active_scope`: `door-to-singularity` и `portal`.

## Потребители команды

- Менять свои workspace-заметки не обязаны, пока не подтягивают канон.
- После подтягивания: при добавлении второго проекта в **door-to-singularity** — новый блок `### Карточка: …` по шаблону из канона.
