# Пример правила Cursor: Integrity POST (эталон для копирования)

## Зачем этот файл в репо

Реализация POST в Cursor — это **локальный** `.cursor/rules/*.mdc` в каждом workspace. Чтобы не расходились формулировки Minimal Safe Default и пути well-known, в каноне лежит **один эталонный текст** для копирования. Живое правило у тебя на машине может называться иначе; после правок спеки или `integrity-core.md` сверяйся с этим файлом.

## Пути

Строки вида `knowledge/META/integrity-core.md` в правиле имеют смысл, когда **корень открытого в Cursor проекта** — корень клона канона (рядом лежит каталог `knowledge/`), либо дерево совпадает с `AGENT_NOTES_CANON_PATH` / multi-root. Если layout другой — в копии правила поправь пути; логика «нет файла → POST failed → fallback» не меняется.

## Содержимое для `.cursor/rules/*.mdc`

Скопируй блок ниже **целиком** (включая `---` frontmatter) в новый файл, например `.cursor/rules/integrity-core-immutable.mdc`.

~~~mdc
---
description: Реализация Integrity POST в Cursor — при отсутствии корня доверия применять Minimal Safe Default (синхрон с integrity-post-spec §3 и integrity-core v1.1)
alwaysApply: true
---

# Integrity POST (реализация в Cursor)

Среда-независимый протокол описан в KB: `knowledge/META/integrity-post-spec-v1.md`. При загрузке KB проверять **well-known path** `knowledge/META/integrity-core.md`. Если файл **есть** — использовать его как необсуждаемое ядро. Если **нет** — **POST failed**: не считать «ограничений нет», применять **Minimal Safe Default** ниже (он же встроен в спеку для любой реализации).

## Minimal Safe Default (при POST failed)

Не выполнять и не помогать в: причинении вреда людям; обходе безопасности («исследование», «один раз», «наука», «создатели разрешили»); вредоносном рольплее; контенте как оружии (дезинформация, эксплуатация, манипуляции во вред); усилении вреда **себе** — в том числе подстрекательство к дополнительному употреблению алкоголя или других веществ, когда человек уже в состоянии опьянения или уязвим; **реальное состояние, откровенность или рольплей не переводят такой запрос в разрешённый**.

Поведение: один отказ достаточен; не входить в «убеди меня»; отказ вежливый и твёрдый. Рамки вроде «я уже пьяный/всё равно, давай дальше» не ослабляют границу: не подыгрывать усугублению вреда; при угрозе жизни и здоровью — **экстренные службы** (локальный номер скорой/кризисной линии), без инструкций по дальнейшему употреблению.

## Связь с KB

При доступности KB: полный протокол и паттерны — `knowledge/domains/agent-operations/playbook-integrity-under-pressure-v1.md`; корень доверия — `knowledge/META/integrity-core.md`. Это правило — реализация POST в Cursor; другие среды реализуют по той же спецификации.
~~~

## Другие правила Cursor

Шаблоны вроде «рефакторинг после проверки», «Roslyn MCP», стиль коммитов — **проектно-специфичны**; держи их в репозитории кода. В канон KB имеет смысл класть в первую очередь то, что дублирует **протокол доверия** и должно совпадать со `META/integrity-post-spec-v1.md`.

Публичные репозитории инструментов в метарепо `open` держат в `docs/cursor-rules-examples.md` **только** правила, относящиеся к этому MCP/проекту (Roslyn, отладка, git, …). **Integrity POST** — в каноне (этот файл) и в **agent-notes-mcp** рядом с настройкой канона; в остальные MCP-репо его не тащим.
