# Checklist: новый устойчивый трек (template)

Использовать, когда появляется **долгоживущее** направление (месяцы+, отдельные артефакты или репо).

- [ ] Выбрать **project-id** (slug, латиница, дефисы)
- [ ] Создать `knowledge/work/projects/<scope>/<project-id>/`
- [ ] Добавить `README.md` по [`template-project-card-v1.md`](template-project-card-v1.md) (минимум: Purpose + Scope + ссылка на репо)
- [ ] Строка в хабе scope (`…/<scope>/<scope>/README.md`)
- [ ] При необходимости — краткая отсылка в `scope-*` в `agent-notes.md` (без длинного дубля)
- [ ] Добавить путь в `work/local/workspace-scope-map-v1.md` (personal)
- [ ] В чате — `[PRIMARY:<project-id>]` по желанию

**Агент:** если пользователь описал трек без карточки — предложить чеклист или явно отложить (черновик в ideas).
