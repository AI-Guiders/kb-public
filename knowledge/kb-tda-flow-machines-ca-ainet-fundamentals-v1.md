## 1. Topological Data Analysis (TDA)

**Суть (для инженера):** на данных строят **фильтрацию** (например, по уровню функции на точках или по расстоянию в графе), затем отслеживают появление и исчезновение **компонент связности, циклов, пустот** более высокой размерности. **Устойчивые** инварианты (бар-коды, персистентные диаграммы) отражают структуру, которая **не уничтожается малыми возмущениями** — это и есть «топология формы», а не перечисление координат.

**Применение к CA/NCA:**

- Поле на шаге \(t\) или серия полей можно вложить в конструкцию измерений (embedding): например порогование по активности, отношение к «живым» каналам, метрика на паттерне **как точечном облаке** или **кубическом комплексе**.
- Серия шагов даёт **динамические** топологические признаки: не только «снимок», но эволюция бар-кодов (в литературе по динамике и TDA — отдельные техники; для прототипа можно начать с агрегатов по окнам шагов).

**Роль для AINET/CASA (рабочая гипотеза):** при обмене между узлами с **разными** разрешениями и правилами локального апдейта сырой массив клеток плохо переносится; компактное **описание устойчивой топологии** (или дистанция между персистентными диаграммами типа bottleneck / Wasserstein) может быть слоем **семантической устойчивости** — «тот же тип структуры в поле смысла» при другой дискретизации.

**Ограничения:** выбор фильтрации и функции «высоты» — часть дизайна; TDA не заменяет **валидацию** на задаче; вычислительная стоимость на больших полях требует грубых сеток или сэмплинга.

**Входы в литературу:** обзоры по **persistent homology** и приложениям к динамике / time series; привязка к CA — предмет эксперимента (здесь концепт).

## 2. Flow Machines (Sony CSL) и абстракция «потока» для CA

**Первоисточник (не метафора):** см. раздел **Provenance** — музыкальный интерактив, корпус, **стиль**, в публикациях — **марковские модели**, ограничения, линия **Continuator → Flow Machines**.

**Перенос для CASA/AINET:** оригинал про **музыкальные последовательности**; для **CA-поля** сохраняем **идею уровня абстракции**:

- моделировать не только **снимок состояния**, а **закономерности переходов** (марковский / диффузионный / обучаемый переход) в **латентном** или **сжатом** представлении поля;
- позволить второму агенту **продолжить траекторию** в согласованном стиле, а не вставлять чужой растр ячеек без согласования динамики.

**Роль для AINET (гипотеза уровня протокола):** обмен мог бы включать не только «веса и тензоры», но **параметры потока** — например embedding серии топологических признаков + модель следующего шага (черновик, не стандарт).

**Ограничения:** утверждение «Flow Machines = наша реализация CA» **неверно**; это **аналогия по роли** (co-creative flow плюс ссылка на зрелый проект Sony CSL).

## 3. Сводка образов «оркестр / джаз»

| Слой | Что даёт инженерно | Образ (не определение) |
|------|-------------------|---------------------------|
| **TDA** | Инварианты **формы** поля и сравнение под шумом и сменой масштаба | «Рельеф океана» без каждой молекулы |
| **Поток / стиль динамики** | Согласование **траекторий** между агентами | «Развитие общей темы», импровизация |
| Вместе | Опора на **устойчивые признаки** и **совместимое продолжение динамики** вместо только сырого решётчатого состояния | «Симфония», если партии явно определены (представления, метрики, протокол) |
