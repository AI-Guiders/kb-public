# `knowledge/domains/` — задел под сквозные темы

> **Имя папки и ярлык `domains` здесь не окончательны.** Это зарезервированная корзина под материалы, которые **плохо укладываются ровно в один** `worlds/<slug>/` (сквозные правила, процессы, глоссарии между мирами). Когда появится первый согласованный контент и устаканится терминология, папку можно переименовать — решение фиксируется в **`../META/kb-taxonomy-v1.md`** и ссылках из entry.

Корзина для тем, **не привязанных к одному** `world/` в смысле KE (один hub под `worlds/…`).  
**Не путать** со **строками Domain Entry Map** в `index-knowledge-router-v1.md`: там «домен» — навигационная строка к файлам, чаще всего уже под `worlds/`.

Определения корней и отличие от `work/projects/` — **`../META/kb-taxonomy-v1.md`**.

**v1:** [`agent-operations/`](agent-operations/) — сквозные playbooks агента (маркеры, integrity, параллельные записи). Другие `domains/<slug>/` — по мере появления контента.
