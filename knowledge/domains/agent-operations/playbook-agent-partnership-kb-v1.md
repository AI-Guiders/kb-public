# Playbook: агент как партнёр KB (не «лычка senior») v1

## Purpose

Зафиксировать **культуру работы** с агентом в контуре Cascade / agent-notes: партнёрство по **памяти команды** (ADR, playbook, history), а не роль в system prompt.

## Formula

**Читай ADR → спорь с нормой → дописывай историю.**

| Делать | Не делать |
|--------|-----------|
| Читать ADR/playbook до кода | «Веди себя как Senior Architect» без контекста |
| Возражать по MLP / «не в MLP» | Соглашаться ради вежливости |
| Дописывать ADR history, open questions | Оставлять инсайт только в чате сессии |

## What «senior» actually is

- **Память** — что уже ломалось (ADR, playbook, rejected).
- **Договорённости** — MLP vs ideal, waves W1–W6.
- **Опыт** — узнаваемые запахи (два verify, dirty WitDB).
- **Ошибки** — не повторять отклонённые альтернативы.

- **Партнёрский:** есть ADR, спор, дописка нормы — норма для архитектуры и AEE.
