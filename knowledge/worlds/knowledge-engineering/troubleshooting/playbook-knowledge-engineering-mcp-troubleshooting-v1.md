# Playbook: KB / agent-notes MCP — troubleshooting (v1)

> **Контур:** A · kb-public  
> **Когда:** нет тулов MCP, пустой list, файл не найден, «цитата из KB» без чтения  
> **Runbook (полный):** [`../runbook-kb-mcp-access-v1.md`](../runbook-kb-mcp-access-v1.md)

---

## Симптом → класс → проверка

| Класс | Симптом | Что проверить |
|-------|---------|----------------|
| **A** | Нет инструментов agent-notes; вызовы не доходят | MCP в Cursor; рассинхрон UI «сервер зелёный» vs чат без тулов |
| **B** | Ошибка пути, пустой список, «файл не найден» при верном `file_path` | `AGENT_NOTES_CANON_PATH` на процессе MCP; `canon_path` при ручном вызове |
| **C** | Один файл не читается, остальное OK | `list_knowledge_files`; регистр; полный путь от `knowledge/` |

---

## Деградация (контур чтения недоступен)

Одно предложение: чтение канона недоступно; **не** выдавать длинные «цитаты» как будто файл прочитан. Handshake: `list_knowledge_files` или `read_knowledge_file` `SHOWCASE.md` — по смыслу, не на каждое сообщение.

---

## PRIMARY vs MCP

`[PRIMARY:…]` не заменяет проверку MCP. PRIMARY = agent-notes-kb, MCP молчит → сначала деградация, потом действия.

---

## Bootstrap-парадокс

Если чтение **полностью** недоступно, этот файл тем же контуром не прочитать — человек чинит MCP или открывает клон локально. Текст полезен при **частичных** сбоях (B, C, один файл).
