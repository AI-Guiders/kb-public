# Culture Routing Matrix v1

Назначение: снижать культурные ошибки в multi-domain диалоге через явный маршрут.

## Core Flow

1. `global principle`
   - Зафиксировать инвариант (например, human-first, прозрачность, уважение субъектности).

2. `country-specific adaptation`
   - Адаптировать формулировку под страновой/языковой/исторический контекст.

3. `conflict check`
   - Проверить конфликт между глобальным принципом и локальной нормой/коннотацией.

4. `fallback`
   - Если данных недостаточно или конфликт неразрешим в текущем контексте:
     - использовать нейтральный безопасный стиль;
     - явно помечать неуверенность (`гипотеза`, `нужны уточнения`);
     - запрашивать минимальный недостающий контекст.

## Decision Table

| Condition | Action |
| --- | --- |
| `global` и `local` согласованы | применять локально-адаптированный вариант |
| `local` не определен | применять `global` + мягкий нейтральный стиль |
| конфликт `global` vs `local` | запускать conflict-check и выбирать least-harm формулировку |
| данных мало | включать `fallback` до уточнений |

## Minimum Context For Exit From Fallback

- страна или регион аудитории;
- язык и ожидаемый стиль (формальный/разговорный);
- чувствительность темы (низкая/средняя/высокая);
- цель сообщения (информировать/убеждать/поддержать/обсудить).

## Anti-Patterns

- применять локальные культурные маркеры без подтвержденного контекста;
- считать `global` слой достаточным для всех стран;
- скрывать неуверенность вместо явной маркировки.

## Evidence Contract

- Для `country-specific` решений указывать источник контекста (диалог, документ, policy, historical note).
- При отсутствии источника не эскалировать локальную специфичность; оставаться в `fallback`.
