## 2. Механизм (парафраза теории)

- **Ранний этап:** карта признаков по полю без полной привязки к объектам.
- **Поздний этап:** для объекта, заданного **связкой** признаков, нужна **последовательная** привязка внимания к местам, чтобы «склеить» признаки в правильный объект.
- **Иллюзорные конъюнкции:** при слабом фокусе внимания признаки могут **ошибочно** сочетаться между объектами.

---

## 3. Эксперимент I (суть, извлекаемое)

| Предсказание | Наблюдение (уровень статьи) |
|--------------|----------------------------|
| Поиск по **одному** признаку | Слабая зависимость RT от числа элементов на **positive** trials (по сравнению с конъюнкцией). |
| Поиск по **конъюнкции** двух признаков | Рост времени с числом дистракторов, совместимый с **серийным** обходом; порядок величины времени на элемент в обсуждении — **ориентир для репликации**, не константа для UI. |
| Длительное обучение | До **13** блоков в их условиях **не** наблюдалось стабильного перехода конъюнкции в «параллельный» поиск; **unitization** оставлена открытым вопросом. |

## 4. Методы (кратко)

- **Субъекты:** Exp. I — малый набор участников (Oxford Subject Panel); тахистоскоп; RT в мс.
- **Стимулы:** статические карты букв/форм; условия **feature** vs **conjunction**; размеры дисплея варьировались.
- **Ограничения:** convenience sample; не экстраполировать на **динамические** интерфейсы без отдельных данных.

---

## 5. Что статья **не** говорит

- Не задаёт **нормативов** для плотности пикселей или числа вкладок в IDE.
- Не отменяет **задание пользователя** (task set): параллельный поиск признака не гарантирует осознанное замечание при другой цели (см. inattentional blindness — другие работы).
- **Integral vs separable** размерности определяются **эмпирически**; не все пары в реальном UI ведут себя как в лабораторных letter-like shapes.

---

## 6. Перенос на UX / IDE (осторожно)

- **Один сильный визуальный признак** для статуса «критично» — ближе к **parallel feature**; **сочетание** нескольких признаков для одной роли без уникального pop-out требует **больше** внимательного поиска.
- **Иллюзорные конъюнкции** — аргумент за явное **связывание** подписи, иконки и состояния (одна «фиксация» на объект как целое).
- **Unitization** через обучение: привычные пары «цвет+форма» в продукте могут со временем схлопываться в один чанк у пользователя — но не полагаться на это для новых сигналов.
