# C# + Roslyn MCP (Cursor agent): диагностики и завершение задачи v1

## Purpose

Зафиксировать обязательный цикл проверки C# через **Roslyn MCP** в среде Cursor, чтобы не пропускать ошибки в **новых или сильно изменённых** `.cs` (типичный промах: проверили один файл, а ошибка — в другом; `dotnet build` показал бы позже).

## Canonical relation

- Общий контекст Roslyn / отладка / сборка: [`dotnet-roslyn-debug-playbook.md`](dotnet-roslyn-debug-playbook.md).
- Этот документ — **узкий протокол** для агента с доступом к `roslyn_*` MCP.

## Rules

### Источник правды для C#

- **Первично:** `roslyn_get_diagnostics` (solution_or_project_path; при необходимости `file_path`).
- **Не считать** ReadLints или разбор `dotnet build` **единственным** источником ошибок C#, если Roslyn MCP доступен.
- **Исправления:** по каждой позиции из диагностик — `roslyn_get_code_actions` → `roslyn_apply_code_action`; не писать патч вручную, если есть применимый code action (fallback на ручное — только если MCP недоступен).

### Обязательно перед тем как считать задачу по C# завершённой

Если в сессии **создавались или существенно менялись** файлы `*.cs`:

1. Вызвать **`roslyn_get_diagnostics`** для соответствующего `.sln` / `.csproj` и **про каждый такой файл** передать `file_path` (как минимум все новые и все сильно затронутые). Пропуск новых файлов — типичная причина «ошибки увидел только из `dotnet build`».
2. Устранить **error** (и по возможности важные **warning**) по цепочке code actions; не сдавать задачу с ошибками компилятора по затронутому коду.
3. Повторный **`roslyn_get_diagnostics`** по тем же путям после исправлений — пока не станет чисто или не останется только согласованный остаток.
4. **`dotnet build`** — дополнительная проверка (MSBuild, весь граф проектов, не-Roslyn шаги), **не замена** пунктов 1–3 для проверки C# «как в VS без полной сборки».

### Где живёт обязательность для Cursor

- **Глобальное User Rule** в Cursor (не workspace): короткая отсылка к этому файлу в каноне + пункты 1–4 в сжатом виде — чтобы правило подмешивалось во всех проектах.
- Детали и обоснование — **только здесь** (канон agent-notes).

## Revisit triggers

- Ошибка обнаружена только при `dotnet build`, хотя файлы уже были в сессии.
- Диагностики вызывали только по одному «главному» файлу.
