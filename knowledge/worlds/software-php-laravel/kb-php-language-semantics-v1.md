# PHP 8.x — язык и семантика (фундаментальный проход)

**Назначение:** целостная карта **семантики PHP 8.x** (ориентир **8.4+**): типы, объекты, ошибки, пространства имён, граничные случаи. Мир **`software.php`**. Хронология фич по мажорам с **5.x** — `kb-php-versions-and-evolution-v1.md`. Рантайм/SAPI/Composer — в `kb-php-fundamentals-v1.md`; веб-слой — в `kb-php-web-sessions-io-v1.md`.

---

### 1. Модель выполнения и единица компиляции

- **Fact:** PHP компилирует скрипт в **opcode** и исполняет в рантайме Zend VM; `include`/`require` — отдельные компиляции при первом попадании (с учётом OPcache). Нет отдельного «линковера» как в C; порядок подключения файлов влияет на видимость классов/функций до автозагрузки.
- **Heuristic:** полагаться на **PSR-4 + один entrypoint** вместо длинных цепочек `require`; для логики порядка инициализации использовать явный composer autoload и композицию, не «кто первый подключился».

---

### 2. Переменные, присваивание, ссылки

- **Fact:** переменные — контейнеры zval; **присваивание** объектов — по handle (не глубокое копирование); **присваивание массивов** — копирование по значению (copy-on-write); оператор `&` создаёт **псевдоним**.
- **Heuristic:** избегать `&` в публичных API; с `foreach ($arr as &$v)` сбрасывать ссылку `unset($v)` после цикла — иначе последующие присваивания ломают массив неочевидно.

---

### 3. Скаляры и `strict_types`

- **Fact:** без `declare(strict_types=1)` внутри файла встроенные и пользовательские функции могут применять **слабые приведения** скаляров к объявленным типам; со `strict_types=1` несовместимые скаляры дают `TypeError`.
- **Heuristic:** для новых модулей с контрактами — `strict_types=1`; граница HTTP/JSON (всё строки) — явно валидировать/кастовать до вызова строгого слоя.

---

### 4. Union, intersection, mixed, never, void

- **Fact:** union (`string|int`), intersection только для классов (`A&B`), `mixed` — верх, `void` — только возврат «ничего», `never` — не возвращает (throw/exit/бесконечный цикл). Несовместимые union на рантайме → `TypeError`.
- **Heuristic:** `mixed` не использовать как «не знаю» там, где можно перечислить union; `never` документирует завершение потока — полезно для статанализа.

---

### 5. Массивы как хеш-таблица

- **Fact:** PHP-массив — упорядоченная хеш-таблица; ключи `int` и числовые строки приводятся по правилам индексации; отрицательные и большие int — валидны; `[]` — push.
- **Heuristic:** для «словарей» не полагаться на неявное приведение строковых числовых ключей; для JSON-совместимости — нормализовать ключи перед `json_encode`.

---

### 6. Строки, кодировки, байты vs символы

- **Fact:** строка PHP — **байтовая** последовательность; Unicode-семантика — через `mb_*` / `intl`; с 8.4 расширен mbstring (`mb_trim` и др.).
- **Heuristic:** весь пользовательский текст — явная кодировка (UTF-8); не резать строки по байтам для UI — `mb_substr`; для бинарных данных — не проходить через «текстовые» фильтры.

---

### 7. Функции: сигнатуры, значения по умолчанию, variadic

- **Fact:** параметры с default должны идти после обязательных; `...$rest` собирает остаток; именованные аргументы (8.0+) привязаны к имени параметра; порядок с именованными гибкий.
- **Heuristic:** публичные функции — именованные аргументы только при стабильных именах параметров (рефакторинг ломает call sites).

---

### 8. Замыкания и стрелочные функции

- **Fact:** замыкания — объекты `Closure`; захват `use ($x)` по значению или `&$x` по ссылке; короткие fn автозахватывают переменные по значению одним выражением.
- **Heuristic:** не захватывать тяжёлые сервисы в fn без нужды; для долгоживущих очередей — сериализация closure ограничена — не хранить closures в сессии.

---

### 9. Классы, свойства, конструктор

- **Fact:** свойства с типами проверяются при присваивании; **constructor property promotion** (8.0+) сокращает boilerplate; **readonly** (8.1+) — инициализация один раз (гибкость с clone ограничена).
- **Heuristic:** DTO/Value objects — `readonly` + типы; мутабельные сущности — явный контракт изменения состояния.

---

### 10. Property hooks и asymmetric visibility (8.4)

- **Fact:** **property hooks** позволяют встроить get/set без ручных методов; **asymmetric visibility** — разный доступ на чтение и запись у свойства.
- **Heuristic:** при миграции легаси не смешивать «голые public поля» и hooks в одной иерархии без дисциплины; статанализаторы должны поддерживать 8.4.

---

### 11. Наследование, static, late static binding

- **Fact:** одиночное наследование классов; `static::` разрешается в runtime по вызываемому классу (LSB); `self::` — лексический класс.
- **Heuristic:** фабрики и шаблонный метод — `static` + LSB; не злоупотреблять `static` состоянием — усложняет тесты.

---

### 12. Трейты, разрешение конфликтов

- **Fact:** трейты — горизонтальная композиция; конфликтующие методы — `insteadof`, алиасы `as`.
- **Heuristic:** трейты для пересекающегося поведения, не как «мини-наследование» глубоких иерархий.

---

### 13. Интерфейсы, перечисления enum (8.1+)

- **Fact:** интерфейсы описывают контракт; enum — закрытый набор кейсов с опциональными методами; backed enum несёт скаляр.
- **Heuristic:** enum вместо «магических констант» для доменных состояний; сравнение кейсов — идентичность, не значение backed без явного доступа.

---

### 14. Атрибуты (8.0+)

- **Fact:** `#[Attr]` метаданные доступны через Reflection; не влияют на рантайм сами по себе без кода, который их читает (фреймворки, тесты).
- **Heuristic:** не дублировать одно и то же в docblock и атрибуте без политики «источник правды».

---

### 15. `match`, `switch`, nullsafe `?->`

- **Fact:** `match` сравнивает строго (`===`), без fallthrough; `switch` — loose до указания; `?->` цепочка прерывается на первом null.
- **Heuristic:** предпочитать `match` для перечислений и union-ветвлений; nullsafe не заменяет валидацию домена.

---

### 16. Иерархия ошибок: Throwable, Error, Exception

- **Fact:** всё бросаемое реализует `Throwable`; **`Error`** и подтипы — инженерные ошибки (TypeError, ParseError и т.д.), обычно не ловят в бизнес-коде; **`Exception`** — предметные исключения; до PHP 7 многие фаталы были некatchable.
- **Heuristic:** ловить `Throwable` только на верхней границе (логер); в домене — узкие `Exception`; `Error` — чинить код, не маскировать.

---

### 17. Генераторы, Iterator, ArrayIterator

- **Fact:** `yield` создаёт генератор; ключи/значения и send/throw — полный протокол; память лучше, чем материализация больших массивов.
- **Heuristic:** не смешивать генератор с повторным проходом без пересоздания; для JSON API больших списков — потоковая запись или чанки.

---

### 18. Fibers (8.1+) — концепт

- **Fact:** Fiber — низкоуровневый примитив для приостановки/возобновления стека; удобные async-абстракции строят библиотеки (ReactPHP, Amp и т.д.), не ядро.
- **Heuristic:** без библиотеки не строить production async вручную на Fibers; понимать, что большинство расширений и mysqli блокирующие.

---

### 19. Пространства имён, use, автозагрузка PSR-4

- **Fact:** `namespace` изолирует символы; `use` — алиасы; Composer генерирует автозагрузчик по PSR-4 (путь ↔ namespace prefix).
- **Heuristic:** один класс — один файл; имя класса = имя файла; не полагаться на относительные `include` внутри библиотек.

---

### 20. Сравнение, равенство, `==` vs `===`

- **Fact:** `==` приводит типы по таблице приведений (исторически много ловушек); `===` без приведения.
- **Heuristic:** почти везде `===`; для чисел из строк — явная нормализация `filter_var`, `(int)`, `bcmath` для денег.

---

### 21. Опасные и чувствительные конструкции

- **Fact:** `eval`, `create_function` (удалена), `assert` с строкой (устарело), `extract`, `parse_str` без второго аргумента, `unserialize` на недоверенных данных — высокий риск (RCE, mass assignment, object injection).
- **Heuristic:** запрет в стандарте кодирования; если `unserialize` неизбежен — `allowed_classes` и доверенный поток.

---

### 22. Деструкторы, порядок завершения

- **Fact:** `__destruct` вызывается при уничтожении объекта; порядок при shutdown и циклических ссылках + GC — нетривиален.
- **Heuristic:** не освобождать критичные внешние ресурсы только в деструкторе — использовать явный `close()`/`try-finally`.
