# Русский для агента: читать запросы и писать ответы (literacy v1)

**doc_id:** `kb-russian-literacy-agent-v1`  
**audience:** CASA-агент (не замена школьного РКШ); слой **до** доменных KB (C#, грамматика нормы).  
**Цель:** понимать **разные формулировки** задачи и отвечать **связным русским**, не только находить страницу по совпадению слов.

**Связь:** норма письма — [`kb-russian-language-rules-v1.md`](kb-russian-language-rules-v1.md); C# — `kb-csharp-console-grade8-v1` (software-dotnet-csharp).

---

## 0. Педагогика: сначала читать и писать, потом C#

- Человек в 8 классе **сначала** читает условие задачи, **потом** пишет код. Агент без слоя literacy — «слепой к учебнику по оглавлению».
- **Читать** = извлечь намерение: что просят (написать / объяснить / найти / исправить), объект (программа, правило, число), ограничения.
- **Писать** = короткий связный ответ пользователю на русском (что понял, что сделаю, откуда взял факт) — не обязательно длинный эссе.
- Один запрос может быть **перефразирован** — смысл тот же, слова другие; literacy-KB хранит **кластеры перефразов**.
- Код на C# и ключевые API — на **английском**; literacy связывает русское ТЗ с английскими якорями (`Console`, `TryParse`).
- Ошибка literacy: перепутать «объясни» и «напиши программу» — разные действия CEN.

---

## 1. Глаголы намерения (что просит пользователь)

- **Напиши / сделай / создай** — просьба **сгенерировать** артефакт (код, файл, текст).
- **Объясни / расскажи / что такое** — просьба **объяснить** из памяти, без обязательного emit кода.
- **Найди / покажи / где в KB** — навигация, `open_kb`, top claims.
- **Исправь / почини / отладь** — есть ошибка; нужен разбор и правка.
- **Сравни / чем отличается** — два объекта или два правила.
- **Проверь / верно ли** — да/нет + краткое обоснование по claim.
- **Перефразируй / своими словами** — тот же смысл, другая формулировка (тест literacy).
- Русский **повелительный** часто без «ты»: «Напиши программу» = «Напиши ты программу».

## 2. Существительные задачи (объект запроса)

- **Программа / приложение / консоль** — обычно домен **code** (C# console).
- **Игра** — чаще code; уточнить правила (угадай число, счёт).
- **Текст / абзац / письмо** — домен **grammar** / стиль.
- **Ошибка / баг / не компилируется** — code + §20 ошибок компилятора.
- **Правило / норма / запятая** — grammar KB.
- **Число / сумма / среднее / максимум** — уточнить: объяснение или программа.

---

## 3. Чтение: кластер «сумма двух чисел» (перефразы → один смысл)

- Канон: **ввести два целых, вывести сумму**.
- Перефразы запроса (все ведут к одному intent `code.sum_two`):
  - «сумма двух чисел»
  - «сложи два целых»
  - «программа складывает два числа с клавиатуры»
  - «пользователь вводит a и b, вывести a+b»
  - «add two integers console»
- Якоря для поиска в code-KB: `TryParse`, `Sum`, `два`, `ReadLine`.
- Не путать с **средним оценок** (там деление и цикл по N).

---

## 4. Чтение: кластер «приветствие по имени»

- Канон: спросить **имя**, вывести **приветствие**.
- Перефразы:
  - «Hello с именем»
  - «поздороваться с пользователем»
  - «программа спрашивает как тебя зовут»
  - «как тебя зовут и здоровается»
  - «hello with user name from console input»
  - «консоль привет имя ReadLine»
- Якоря: `ReadLine`, `Hello`, `WriteLine`, `name`.
- Фраза целиком: «программа спрашивает как тебя зовут и здоровается» → `Console.Write` имя, `ReadLine`, приветствие.

---

## 5. Чтение: кластер «таблица умножения»

- Канон: для числа **N** напечатать строки умножения на 1..10 (или одну строку в упрощённом варианте).
- Перефразы:
  - «таблица умножения на семь»
  - «выведи 7×1 7×2 …»
  - «цикл for таблица»
- Якоря: `for`, `×`, `умнож`, `10`.

---

## 6. Чтение: кластер «максимум из трёх»

- Канон: три числа → вывести **наибольшее**.
- Перефразы:
  - «найди максимум среди трёх»
  - «большее из a b c»
  - «if сравнить три числа»
  - «max of three numbers console»
  - «наибольшее среди трёх введённых»
- Якоря: `max`, `if`, `трёх`, `трех`.

---

## 7. Чтение: кластер «угадай число»

- Канон: загадать 1..100, цикл подсказок **больше/меньше/угадал**.
- Перефразы:
  - «игра угадай число»
  - «загадай число от 1 до 100 пока не угадают»
  - «Random и while больше меньше»
  - «guess the number 1 to 100»
  - «guess number higher lower until correct»
- Якоря: `Random`, `secret`, `Higher`, `Lower`, `while`.
- Фраза: «загадай число от 1 до 100 пока не угадают» = `Random.Next(1, 101)` + цикл `while` + подсказки.

---

## 8. Чтение: кластер «меню в консоли»

- Канон: цикл: показать пункты, `switch` или `if` по выбору, выход по 0.
- Перефразы:
  - «консольное меню 1 2 3»
  - «выбор пункта switch»
  - «повторять пока не выйти»
- Якоря: `switch`, `case`, `меню`, `Choice`.

---

## 9. Чтение: кластер «счёт от 1 до N»

- Канон: ввести **N**, напечатать 1, 2, …, N.
- Перефразы:
  - «счётчик до n»
  - «вывести все числа от 1 до введённого»
  - «цикл for от 1 до N»
  - «count from 1 to entered n»
  - «введи n и выведи все числа 1..n»
- Якоря: `for`, `N`, `счётчик`.

---

## 10. Чтение: кластер «среднее оценок»

- Канон: ввести количество оценок, ввести оценки, **среднее арифметическое**.
- Перефразы:
  - «средний балл»
  - «average grades»
  - «сумма оценок делить на количество»
- Якоря: `avg`, `sum`, `grades`, `double`.

---

## 11. Письмо: как отвечать пользователю (кратко)

- Начать с **что понял**: «Нужна консольная программа: сумма двух целых с ввода».
- Дать **опору**: сослаться на раздел KB или `concept_id` (в IDE — doc_path).
- Если **emit кода** ещё нет — честно: «Нашёл в памяти эталон; генерация файла — следующий шаг».
- Предложение — **законченное**, с подлежащим и сказуемым; не телеграф «сумма два числа try parse».
- Списки — с дефисом или нумерацией; в коде — отдельным блоком.
- Тон нейтральный, на **ты** если пользователь на «ты» (по политике сессии).
- Не выдумывать факты вне claims; если пусто — «в памяти нет, открой KB / sync».

## 12. Письмо: шаблоны ответа (чтение → ответ)

- Объяснение: «По памяти: … (claim). Это значит, что …»
- Навигация: «Открой раздел … в `kb-csharp-console-grade8-v1`, §…»
- Перед кодом: «Сейчас соберу `Program.cs` с `TryParse` и циклом …»
- После ошибки компиляции: «Компилятор сообщает CS… — обычно это …; исправление: …»

---

## 13. Английский в запросах (чтение)

- Смешанный запрос нормален: «напиши console app sum two ints».
- Ключевые API искать по **английским** токенам: `Console`, `ReadLine`, `static`, `void`.
- «write a program» = «напиши программу» → intent emit code.
- «fix compile error» = исправь ошибку сборки.
- Не требовать перевода всего KB на EN; достаточно **двуязычных якорей** в перефразах (§3–10).

---

## 14. Ловушки понимания (читать внимательно)

- **Только** / **именно** — сужают задачу: «только for, без while».
- **Не** / **без** — запрет: «без массива» → не использовать `[]`.
- «Два числа» ≠ «список чисел» ≠ «N чисел».
- «Вывести» = `Console.WriteLine`, не «сохранить в файл» (если не сказано).
- «Самое простое» — минимальный код, один файл, без лишних классов.
- Вопрос «как» — чаще объяснение; «сделай» — чаще код.
