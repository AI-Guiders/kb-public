# Playbook: nginx — администрирование (v1)

> **Scope:** типовые настройки nginx, работа с PHP-FPM и proxy. Без привязки к конкретным доменам и путям.

---

## Ключевые понятия

- **client_max_body_size:** максимальный размер тела запроса (по умолчанию 1M). При превышении — 413. Задаётся в http, server или location.
- **FastCGI / PHP-FPM:** nginx передаёт запросы приложению через FastCGI; PHP-FPM — менеджер процессов PHP. Связь через Unix-сокет или TCP.
- **stub_status:** модуль nginx для выдачи метрик (соединения, запросы) — для мониторинга (Zabbix и др.).

---

## Практики

- **Права и пользователи:** отдельный пользователь/пул PHP-FPM на приложение; nginx под своим пользователем; nginx добавлен в группу владельца файлов сайта для чтения. Владелец файлов — пользователь приложения, не nginx. Избегать 777 и владения nginx:nginx на файлах приложения.
- **Связь с PHP-FPM:** предпочтительно Unix-сокет (`fastcgi_pass unix:/path/to/socket`); быстрее и без сетевого стека.
- **Безопасность:** заголовки X-Frame-Options, X-Content-Type-Options, X-XSS-Protection; запрет доступа к скрытым файлам `location ~ /\. { deny all; }`.
- **Статика:** отдавать статику из nginx, кэшировать; не проксировать её в PHP без необходимости.

---

## Типовая проблема: 413

- В нужном server/location задать `client_max_body_size 50M;` (или иное значение по требованиям).
- Проверка: `nginx -t`, затем `nginx -s reload`.

---

## Анти-паттерны

- Общий пользователь для всех сайтов (www-data на всём) — смешение прав и рисков.
- Reload без `nginx -t`.
- Отсутствие лимита на размер тела при загрузке файлов — риск DoS и 413 без явной настройки.
