# Playbook: Wireshark — диагностика сети (v1)

> **Scope:** общие приёмы захвата и анализа трафика. Без привязки к конкретным сетям и хостам.

---

## Ключевые понятия

- **Capture filter (фильтр захвата):** задаётся до начала захвата, отсекает пакеты на уровне сбора (синтаксис libpcap/tcpdump). Сокращает размер дампа.
- **Display filter (фильтр отображения):** применяется к уже захваченным данным; можно менять без нового захвата. Синтаксис Wireshark (например `tcp.port == 80`).
- **Follow stream:** просмотр полного потока запрос/ответ по одному соединению.

---

## Типовые фильтры захвата

- По хосту: `host 192.168.1.1`; по подсети: `src net 192.168.0.0/24`.
- По порту: `port 53` (DNS), `port 80` или `port 443` (HTTP/HTTPS).
- Исключение: `port not 53 and not arp`.
- Только unicast: `not broadcast and not multicast`.
- Только IPv4: `ip`.
- Сбросы TCP (таймауты): в display filter `tcp.flags.reset == 1`.

---

## Типовой сценарий диагностики

1. Выбрать нужный интерфейс и начать захват.
2. Воспроизвести проблему во время захвата.
3. Остановить захват, применить display filter к интересующему трафику.
4. Использовать Follow Stream для анализа диалогов.
5. Сохранить захват в .pcap для передачи или архива.

---

## Анти-паттерны

- Захватывать весь трафик без фильтра на перегруженных сегментах — большой объём и шум.
- Путать capture и display filter — первый уменьшает объём дампа, второй только отображение.
- Хранить или передавать дампы с конфиденциальными данными без ограничения доступа.

---

## Диагностика

- При «ничего не видно» — проверить интерфейс (promiscuous mode), наличие трафика на данном сегменте, корректность фильтра.
- При анализе задержек — смотреть на delta времени между пакетами, ретраны и RST.
