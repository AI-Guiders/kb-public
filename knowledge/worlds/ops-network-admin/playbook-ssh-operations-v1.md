# Playbook: SSH / SCP — операции и безопасность (v1)

> **Scope:** общие практики использования SSH, SCP и управления ключами. Без конкретных имён хостов и путей.

---

## Ключевые понятия

- **Аутентификация по ключу:** пара ключ (приватный/публичный); приватный ключ хранится у пользователя, публичный — на сервере в `~/.ssh/authorized_keys`.
- **SCP:** копирование файлов поверх SSH (`scp [опции] источник целевой_хост:путь`).
- **Bastion (jump host):** промежуточный хост для доступа во внутреннюю сеть; уменьшает площадь атаки.
- **SSH Agent Forwarding:** использование локального агента для аутентификации при проходе через bastion без копирования ключей на промежуточный хост (риск: при компрометации bastion возможен перехват сессии).

---

## Рекомендации по ключам

- Предпочтительно **Ed25519**; при необходимости RSA — не менее 4096 бит. Приватный ключ защищать паролем.
- Приватные ключи не копировать на другие машины; для доступа через bastion использовать agent forwarding или отдельные ключи с минимальными правами.
- В production отключать вход по паролю, использовать только ключи. Для автоматизации — отдельные ключи с ограниченными правами, по возможности краткосрочные.

---

## Bastion и сеть

- Bastion держать с минимальным набором сервисов; неправильная настройка сети может позволить обойти bastion — важен контроль маршрутов и firewall.
- Применять принципы zero trust: соединения с bastion дополнительно проверять (например, по сертификатам или MFA).

---

## Анти-паттерны

- Хранить приватные ключи в репозиториях или на общих дисках.
- Оставлять парольный вход включённым на внешних серверах.
- Использовать один долгоживущий ключ для всех целей (человек и автоматизация).
