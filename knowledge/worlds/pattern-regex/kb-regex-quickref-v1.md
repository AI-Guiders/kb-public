# Regex: шпаргалка (L0)

## Назначение

Минимум символов для чтения и набора паттерна; детали — в `kb-regex-syntax-features-v1.md`.

## Метасимволы (типичный POSIX/PCRE/.NET стиль)

| Класс | Символы | Смысл |
|--------|---------|--------|
| Литералы | буквы/цифры (часто) | Сами себе |
| Точка | `.` | Один символ (кроме перевода строки — зависит от режима) |
| Класс | `[...]`, `[^...]` | Один символ из набора / не из набора |
| Квантификатор | `*`, `+`, `?`, `{m,n}` | Повтор предыдущего «элемента» |
| Группа | `(...)` | Капчур + приоритет + квантификация блока |
| Или | `\|` | Альтернатива |
| Якорь | `^`, `$` | Начало/конец строки (зависит от многострочности) |
| Граница слова | `\b`, `\B` | Между `\w` и не-`\w` |

## Частые шорткаты классов

| Шорткат | Обычно |
|---------|--------|
| `\d` | Цифра |
| `\w` | «Словесный» символ (зависит от Unicode-режима) |
| `\s` | Пробельный |

## Экранирование

- В regex-литералах языка и в разных инструментах число обратных слэшей разное: **всегда** сверяй с правилами строки языка (C# verbatim `@"..."`, JS template vs обычная строка).

## Жадность

- По умолчанию квантификаторы **жадные** (берут максимум, отдают назад при необходимости).
- `*?`, `+?`, `??`, `{m,n}?` — ленивые (минимум совпадения).

## Три вопроса перед матчем

1. Какой **диалект** (JS, .NET, PCRE, grep -E)?
2. Нужен ли **многострочный** или **однострочный** режим для `^$` и `.`?
3. Есть ли **таймаут** или лимит шагов в проде (.NET `MatchTimeout`)?
