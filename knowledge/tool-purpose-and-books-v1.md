# Назначение технологий, роутинг по задачам и книги для углубления БЗ (v1)

> **Назначение:** (1) ориентир «для чего какая технология»; (2) роутинг «задача → инструмент», чтобы при появлении аналогов выбирать нужный инструмент; (3) список из трёх книг по каждой изученной теме для последующей загрузки в БЗ.

---

## 1. Назначение технологий (ориентир)

Кратко — **для чего** каждая из изученных тем используется, чтобы не путать роли и при добавлении аналогов понимать, что с чем сравнивать.

| Технология / область | Основное назначение | Что даёт на практике |
|----------------------|---------------------|------------------------|
| **Zabbix** | **Мониторинг инфраструктуры и алертинг** | Сбор метрик с хостов (агенты, SNMP, HTTP, JMX), хранение истории, триггеры и уведомления при сбоях, карты, IT‑сервисы и SLA. «Узнать, что сломалось и когда». |
| **Grafana** | **Визуализация и дашборды** | Подключение к разным источникам данных (Prometheus, Loki, InfluxDB, Zabbix через плагин и т.д.), построение графиков и панелей, алерты по запросам. «Красиво увидеть и показать метрики/логи/трейсы». Сама метрики не собирает — только запрашивает и рисует. |
| **Сетевые технологии (fundamentals)** | **Фундамент для диагностики и проектирования сетей** | Модель OSI/TCP/IP, устройства (коммутатор, роутер, файрвол, прокси), протоколы (IP, TCP, UDP, DNS, DHCP, VLAN, STP и т.д.). Нужны для понимания «почему не ходит трафик», настройки сетей и выбора решений. |

**Итог:** Zabbix — про **мониторинг и алерты**; Grafana — про **графики и дашборды** (и алерты поверх уже собранных данных). Сети — база, без привязки к конкретному софту.

## 2. Роутинг по задаче → инструмент (и где в БЗ)

Когда встаёт вопрос «какой инструмент использовать» (в т.ч. после добавления аналогов), опираться на эту таблицу. В колонке «В БЗ» указаны домены и файлы в agent-notes.

| Задача | Подходящие инструменты (изученные / к добавлению) | В БЗ (домен / файлы) |
|--------|----------------------------------------------------|----------------------|
| Сбор метрик с серверов и сетевых устройств, алерты при порогах | **Zabbix**, Nagios, Prometheus + Alertmanager, Datadog, PRTG… | zabbix-monitoring → kb-zabbix-reference-v1, playbook-zabbix-monitoring-v1 |
| Дашборды и графики по уже собранным данным | **Grafana**, Zabbix (встроенные графики/карты), Kibana, Datadog UI… | grafana → kb-grafana-reference-v1, playbook-grafana-v1 |
| Логи (сбор, поиск, визуализация) | Loki + Grafana, ELK (Elasticsearch, Logstash, Kibana), Splunk… | grafana (Loki как data source); отдельные домены при добавлении ELK/Splunk |
| Трейсы (распределённая трассировка) | Tempo + Grafana, Jaeger, Zipkin… | grafana (Tempo как data source) |
| Метрики + алерты в одном продукте «из коробки» | **Zabbix**, Nagios, PRTG | zabbix-monitoring |
| Только визуализация поверх любых источников | **Grafana** | grafana |
| Понимание сети, диагностика (ping, трассировка, VLAN, STP) | Не «инструмент», а знания по протоколам и устройствам | network-fundamentals → kb-network-reference-v1, playbook-network-fundamentals-v1 |

**Правило для агента:** при вопросе «что выбрать для мониторинга» или «Zabbix vs Prometheus vs Grafana» — сначала открыть этот файл и domain-index-v1.md; по задаче пользователя определить строку таблицы (сбор метрик / дашборды / логи / трейсы) и подставить изученные инструменты; при наличии аналогов в БЗ — сравнить по соответствующим kb/playbook.

Дополнительный роутинг по ключевым словам запроса остаётся в **domain-index-v1.md** (раздел «Роутер: тип задачи → домен»). Этот файл дополняет его смыслом «для какой задачи какой класс инструментов».

## 3. Книги для углубления БЗ (три на тему)

После того как пользователь «скормит» эти книги агенту, ожидается доработка kb и при необходимости playbook до исчерпывающей полноты по каждой теме.

### Zabbix (мониторинг)

1. **Zabbix 7 IT Infrastructure Monitoring Cookbook, 3rd Edition** — Nathan Liefting, Brian van Baekel (Packt, 2024). Актуальные возможности Zabbix 7: установка, масштабирование, облака (AWS, Azure, Docker), триггеры и алерты, HA, API, интеграции (Slack, Teams, OpsGenie), БД и бэкапы. Подходит и для старта, и для углубления.
2. **Zabbix 6 IT Infrastructure Monitoring Cookbook, 2nd Edition** — Nathan Liefting, Brian van Baekel (Packt). Рецепты по Zabbix 6; многое применимо к 7. Дополняет первую книгу практикой.
3. **Zabbix 4 Network Monitoring** — Patrik Uytterhoeven (Packt). Классика по основам: SNMP, VMware, JMX, диагностика. Базовые концепции (хосты, шаблоны, item'ы, триггеры) мало меняются между версиями — полезна для закрепления фундамента.

### Grafana (визуализация и дашборды)

1. **Learn Grafana 10.x, 2nd Edition** — Eric Salituro (Packt, 2023). От установки до продвинутых дашбордов: источники данных (Prometheus, InfluxDB, Elasticsearch), переменные, алертинг, аутентификация. Удобно для систематического покрытия возможностей Grafana.
2. **Observability with Grafana** — Rob Chapman, Peter Holmes (Packt, 2024). Стек LGTM (Loki, Grafana, Tempo, Mimir): логи, метрики, трейсы, дашборды, алерты, Kubernetes и облака (AWS/GCP/Azure), SLO/SLI, Faro, Pyroscope, k6. Расширяет БЗ от «графиков» до полной наблюдаемости.
3. **Grafana 10.x** или аналог по версии (при наличии отдельной книги по администрированию/enterprise) — либо вторая книга из списка выше плюс официальная документация. Резерв: повторное использование разделов «Learn Grafana 10.x» по админ-функциям (provisioning, роли, LDAP) как третьего источника.

### Сетевые технологии (основы)

1. **CCNA 200-301 Official Cert Guide, Volume 1, 2nd Edition** — Wendell Odom (Cisco Press, 2024). TCP/IP, Ethernet, IP-маршрутизация, коммутация, базовые темы CCNA. Надёжный структурированный фундамент.
2. **CCNA Certification Study Guide Volume 1, 2nd Edition (Exam 200-301 v1.1)** — Todd Lammle, Donald Robb (Wiley, 2024). Альтернативный разбор тем CCNA, практика и подготовка к экзамену — дополняет Odom другим изложением.
3. **Acing the CCNA Exam, Volume 1: Fundamentals and Protocols** — Jeremy McDowell (Manning, 2024). Основы и протоколы в сжатом виде; удобно для закрепления и быстрого поиска по темам.
