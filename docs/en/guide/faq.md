# Quick-entry FAQ

## Why KB if we already have code and chat?

So decisions survive sessions and you do not repeat the same explanations.

## What is the source of truth?

Decisions and protocols — `agent-notes` (KB / this site). Implementation — code in product repos.

## When to write to KB?

When the decision is reusable, reduces errors, and helps someone later.

## When not to write?

One-off daily noise with no reuse.

## I lost context. Where to start?

Home → entry KB → current goal → last recorded step → one verifiable step.

## Too many tasks?

Keep one active focus; freeze others with return anchors.

## Are WORK and HUMAN mandatory?

Yes, if you want communication quality under load. Marker tables, `[PRIMARY]`/`[SCOPE]` — [Protocols](../onboarding/protocols.md).

## What is soft reject?

Temporarily defer a HUMAN request due to urgency, with mandatory return.

## How do I know I entered the environment?

You can name source of truth, restore context, and pick the next verifiable step without hints.
