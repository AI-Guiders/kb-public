# Typical scenarios and recovery

Quick decisions when cognitive load is high.

## Scenario 1: Lost in a task

Signs:

- lots of reading, little movement;
- unclear next step.

Recovery:

1. Pause 2 minutes.
2. State the goal of this pass (one sentence).
3. Pick **one** output artifact.
4. Do the minimal step and verify.

## Scenario 2: Too many parallel threads

Signs:

- constant switching;
- no cycle closes.

Recovery:

1. Freeze secondary branches.
2. One active focus for 25–40 minutes.
3. Return anchors for the rest:
   - where stopped,
   - next step,
   - blocker.

## Scenario 3: Production incident

Signs:

- priority shifts to incident mode.

Recovery:

1. Switch to WORK.
2. State soft reject (if a HUMAN thread was open).
3. Close the urgent block.
4. Return and reopen the HUMAN thread.

## Scenario 4: Do not understand the environment

Recovery:

1. Return to [Entry KB](../knowledge/00-entry-kb-v1.md) and [home](../index.md).
2. Check [Memory and KB](memory-and-kb.md).
3. Formulate three questions:
   - context,
   - where stuck,
   - what you already tried.

## Scenario 5: Fatigue and cognitive drop

Recovery:

1. Shrink planning horizon to one step.
2. Do the cheapest verifiable step.
3. Record state and stop cleanly.

## Mini-rule

When thinking is hard, do not complicate the system — shrink the step to a verifiable minimum.
